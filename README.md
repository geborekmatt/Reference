# Reference
A curated, opinionated source of truth for how I build software.

## Overview
This repository contains production-ready patterns, snippets, and architectural decisions that reflect my preferred way of writing code across projects.

It serves two primary purposes:

1. Personal Reference
   * Eliminate repeated decisions
   * Reuse proven patterns
   * Maintain consistency across projects
2. AI Alignment Layer
   * Provide structured context to LLMs
   * Normalize generated code to match my standards
   * Reduce inconsistent or low-quality outputs

## Philosophy
Every entry in this repository is:

* Opinionated – reflects a deliberate choice, not multiple options
* Production-ready – usable in real applications
* Consistent – follows shared patterns across the codebase
* Minimal but complete – no unnecessary abstraction or fluff

## Pattern Format
Each pattern should follow a consistent structure:

### Code Files
* Fully working implementation
* Includes comments explaining intent
* Designed to be copied into real projects

### Markdown Patterns
```md
# Pattern: Example Name

## When to Use
- Situation 1
- Situation 2

## Why This Approach
- Key reasoning
- Tradeoffs considered

## Implementation
(code)

## Gotchas
- Common mistakes
- Edge cases
```
